# Sprint 2.2 — Memory Intelligence Pipeline 实现计划

> **Goal:** 从用户输入中智能提取结构化记忆，经过意图检测→提取→验证→去重→溯源五阶段 Pipeline

## Pipeline 流程

```
用户输入
  ↓
Intent Detection    → 判断是否包含可提取记忆
  ↓
Memory Extraction   → LLM 提取结构化记忆
  ↓
Memory Validation   → 置信度评分 + 来源类型
  ↓
Memory Deduplication → 已有记忆相似度检查
  ↓
Memory Storage      → 写入 persona_memories + memory_sources
```

## 文件结构

```
backend/app/ai/
├── services/
│   ├── memory_intelligence_service.py   ← Pipeline 主服务 (NEW)
```

### Task 2.2.2: Memory Extraction（增强版）

**目标:** 从文本中提取结构化记忆，输出包含置信度和来源类型的完整结构

- `source_type`: "direct_statement"(直接陈述) / "inference"(推断) / "observation"(观察)
- `confidence`: 0.0-1.0
- 同时提取可能的 Event（如生日、纪念日）

---

### Task 2.2.3: Memory Validation + Deduplication + Provenance

**目标:** 
- Validation: 过滤低置信度 (<0.5) 的记忆
- Deduplication: 检查已有记忆中相似条目，相近则合并而非新增
- Provenance: 写入 memory_sources 表记录来源

Deduplication 策略（V1 简单版）：
1. 相同 persona_id + 相同 category
2. content 关键词重叠度 > 60% → 视为重复
3. 重复时：更新已有记忆的 importance，不新增
